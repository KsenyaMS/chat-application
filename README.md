# Описание
1. Взять дизайн проект из фигмы.
2. Использовать для верстки Mantine. Настроить светлую и темную тему.
3. Использовать css-модули.
4. Использовать redux для создания хранилища. Выяснить, можно ли использовать одно хранилище для разных частей приложения.
5. Список пользователей для регистрации/авторизации.
Поля:
1. userName,
2. password,

6. Детальная информация о пользователе.
Поля:
1. ID,
2. userName,
3. email,
4. phone,
5. chats: [{id, messages: [], friend: {}}],
6. newMessages: [{id, count}]

7. Использовать react form hook,
8. Аватарки,
9. Добавить дизайн диалогов,
10. Добавить все виды тестов,
11. Использовать Yup или zod.
12. Настроить eslint, prettier, typescript.
13. Использовать vite.
14. Посмотреть, получится ли использовать базу с тестовыми данными, если выйдет, решить, какие запросы будут браться. Как вариант - axios.

15. Использовать правильный подход к архитектуре проекта.


Что я хочу видеть в своем проекте:
1. Форма регистрации/авторизации в том же стиле, что и проект.
2. На фоне - картинка.
3. Никаких верхних меню и т.п., только основной блок в середине. Блок никогда не будет меняться. Меняться будет контент.
4. После авторизации пользователь видит список своих диалогов.
В списке есть аватарки пользователей, логины, отметки о новых сообщениях.
5. Также есть общий список пользователей/друзей. Там только аватарки и логины.
6. Пользователь может перейти в личный чат и посмотреть историю своих сообщений.
Там будут сообщения, поле для отправки сообщения, аватарки обоих пользователей. Время отправки сообщений.
7. Еще будет кнопка смены темы.
